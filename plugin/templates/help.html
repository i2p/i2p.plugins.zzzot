Welcome to the ZzzOT I2P Plugin!

This help file is located at: $PLUGIN/eepsite/docroot/help.html
You should probably move it outside of the document root before you announce your eepsite as it may contain your username.

ZzzOT is a simple in-memory BitTorrent open tracker, wrapped into an I2P plugin. The software depends on several I2P libraries, in addition to i2psnark.jar from the I2P installation for bencoding. Please report bugs on i2pforum.i2p. New releases will be announced on the plugin forum on zzz.i2p.

Source code is available (under the Apache 2.0 license) at our Gitea site and on github. Note that the I2P source code must be available (and compiled) in ../i2p.i2p in order to build ZzzOT.

Configuration & Customization

  • A new eepsite tunnel and Jetty server have been started for your open tracker, configured for 2 inbound and 2 outbound tunnels, 3 hops each. If your tracker gets over 1000 peers, you will probably want to increase the number of tunnels.
  • To change the tunnel settings, edit: $PLUGIN/i2ptunnel.config (the tunnel will not appear in the Tunnel Manager), then stop and restart the plugin from the Plugin Manager.
  • The Jetty webserver is running on port 7662. If you must change it, edit: jetty.xml, i2ptunnel.config, and plugins.config in: $PLUGIN/, then stop and restart the plugin.
  • The configuration file for ZzzOT is located at: $PLUGIN/zzzot.config
  • The most important configuration for ZzzOT is the interval, which tells clients how long to wait between announces. The default is 1620 seconds (27 minutes).
  • The easiest way to reduce the load on your tracker is to increase the interval. To do so, edit the configuration file, then stop and restart the plugin.
  • UDP may also be enabled or disabled, and the connection lifetime changed, in the configuration file.
  • All conguration changes require the plugin to be stoped and restarted.
  • Live stats are available on the tracker page (this link is also on your router console sidebar when ZzzOT is running).
  • To change the display name for the site (for the logo and page titles), add the line: sitename=mytracker to the configuration file (substituting mytracker with your desired name) and then stop and restart the plugin.
  • To change the footer text displayed on the tracker stats page, add the line: footertext=alternative text to the configuration file, and then stop and restart the plugin.
  • To hide the version footer displayed on the stats page, add the line: showfooter=false to the configuration file, and then stop and restart the plugin.
  • Your eepsite document root is: $PLUGIN/eepsite/docroot/. You may put other files there if you wish to host additional content on your eepsite.
  • Your eepsite home page is located at: $PLUGIN/eepsite/docroot/index.html, you may edit it to add information about your tracker or links to additional content, or to change the displayed name and page title.
  • If you wish to change the appearance of the site, edit the css file located at:
    $PLUGIN/eepsite/docroot/tracker.css. There's an alternative purple-themed css file in the same directory that should be renamed to tracker.css (having renamed the existing file) if you wish to use that instead, or deleted if you don't intend to use it.
  • Directory listings are enabled on the server, so you don't need to create a web page to serve files; content in any folders you create in your document root will be immediately available to your visitors. The quickest method to disable listings is to place an empty index.html file in any folder you wish to disable.
  • Open trackers do not require torrents to be registered, and do not host torrent files. However, you can host torrent files elsewhere on the eepsite, for example at /torrents.

Eepsite Key & Hostname Registration

Your private key file is located at: $PLUGIN/eepPriv.dat
Make sure you backup this file and move the copy outside of $PLUGIN/ as it controls ownership of your B64/B32 address.

Your Base 32 address is: $B32
Others may access your eepsite using this address, even if you do not publish a hostname.

Your Base 64 key is:
$B64
Once you decide on a hostname, you may add the key to your local addressbook.

To generate a registration authentication string for your hostname (required for hostname registration at stats.i2p), run the following from the command line:
java -jar $I2P/lib/i2p.jar privatekeyfile -a your-hostname.i2p $PLUGIN/eepPriv.dat
